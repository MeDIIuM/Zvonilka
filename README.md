# Звонилка (MVP)

Минимальное iOS-приложение с одним экраном:
- список контактов
- сортировка по количеству исходящих (по нажатию "Позвонить")
- быстрый звонок по первому номеру контакта

## Что реализовано
- `Contacts` permission + чтение контактов
- Фильтрация контактов без номера
- Локальный счетчик исходящих в `UserDefaults`
- Сортировка: сначала по счетчику (убывание), затем по имени
- Поиск по имени и номеру
- Инициация звонка через `tel://`
- Кнопка `Системный выбор` с нативным `CNContactPickerViewController`

## Запуск и тест на ноутбуке (без iPhone)
Для быстрого тестирования UX/логики добавлен desktop preview:
- `LaptopPreview/index.html`
- `LaptopPreview/app.js`
- `LaptopPreview/styles.css`

Как запустить:
1. Откройте `LaptopPreview/index.html` в браузере.
2. Используйте поиск и кнопки "Позвонить".
3. После каждого нажатия счетчик увеличивается и контакт поднимается выше в списке.

Что в preview повторяет MVP:
- Один экран и один список.
- Сортировка по исходящим нажатиям.
- Инкремент счетчика по факту нажатия.
- Звонок по первому номеру контакта.
- Локальное хранение счетчиков через `localStorage`.

Примечание:
- На ноутбуке `tel:` может не запускать реальный звонок, это нормально.
- Для полной проверки системного доступа к контактам и реального вызова нужен запуск iOS-версии на Mac/Xcode.

## Ограничения MVP
- Нет записи звонков
- Нет облачной/серверной синхронизации
- Нет аналитики
- Нет учета входящих
- Нет разделения личных/рабочих
- Потеря локальных данных допустима

## Важные технические моменты
- Инкремент счетчика происходит в момент нажатия "Позвонить".
- Результат звонка не отслеживается.
- При нескольких номерах берется первый доступный номер.

## Интеграция в Xcode
1. Создайте iOS App-проект (SwiftUI).
2. Добавьте файлы из `Sources/` в target приложения.
3. Замените `Info.plist` на `Resources/Info.plist` или добавьте `NSContactsUsageDescription` вручную.
4. Запустите на iPhone (симулятор не умеет реальный звонок).
